# Contributing

Solo learning project using GitHub Flow:

1. Branch off `main`: `feature/…`, `fix/…`, or `chore/…`
2. Commit as you go
3. Push and open a pull request into `main`
4. Wait for the CI check to pass (syntax/import check — see `.github/workflows/ci.yml`)
5. Merge via "Squash and merge", then delete the branch

`main` is protected — no direct pushes, even for the repo owner.
